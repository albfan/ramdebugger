Known bugs

  • TCL code that returns a value without using return, will fail when being debugged
  • Workaround: Use always return if you want to return a value from a proc
  • Lines of code inside an expression cannot be debugged
  • Workaround: Instead of:

if { [catch {

call_proc1

call_proc2

...

} errstring] } {

...

}

Use:

set err [catch {

call_proc1

call_proc2

...

} errstring]]

if { $err } {

...

}
